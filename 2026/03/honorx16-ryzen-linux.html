    • 荣耀 X16 锐龙版（2024）运行 Linux 问题和修复

    下一台电脑不会再买荣耀的了

     March 28 技术探索 785 字
    智能摘要
    Qwen 3.7 Max
    作者 2024 年大一因专业需求与性价比，入手国补后约 2.8k 的荣耀 X16 锐龙版笔记本，前期使用 Windows 系统表现正常，年底安装 CachyOS Linux 后接连出现问题：一是设备睡眠唤醒极慢甚至无法唤醒，源于电脑 BIOS 阉割 S3 睡眠模式仅支持 S0ix，后通过 UAFB 开启 AMD RAID Mode 得以解决；二是 HDMI 外接显示器颜色失真，Linux 系统内色彩范围设置无效，改用 USB-C 接 Steamdeck 扩展坞以 DP 输出后恢复正常。作者据此不建议 Linux 用户选购新款荣耀电脑，因其 BIOS 阉割严重导致问题难解决，老款荣耀 Magicbook 则无此类问题，同时该电脑在 Windows 系统下优化表现良好。

    前言

    在 2024 年要上大一的时候，我决定更换一台笔记本电脑，当时考虑个人专业并不是什么图形设计或者视频剪辑以及工业绘图什么的，所以没打算整多高配置要求的电脑，也是考虑给家里减轻下负担，毕竟花的是父母的钱。

    但是也不能随便挑台花瓶机器，当时 Intel 的 CPU 性价比评价不如 AMD 的 Ryzen 系列，所以打算入手一台锐龙机器，看了下综合评价后选择了荣耀的 X16 锐龙版（16+512G,R5-7640HS），国补后大概 2.8k 的价格还是不错的。

    然后一直到 24 年底之前我都是主要用 Windows，大概到年底接触了 Linux，于是在电脑上安装了 CachyOS，之后就出了各种各样奇怪的问题至今，虽然现在算是“解决了”，但都是治标不治本，简单记录下问题和解决的办法，供以后参考和评判。

    设备睡眠后唤醒速度极其慢

    当时在使用的时候，遇到了合上本子睡眠后再唤醒非常慢的问题，大概从黑屏到唤醒需要 8s 以上，甚至有时候可能就这么睡死了，完全无法唤醒。

    简单检查后发现荣耀的 BIOS 阉割了 S3 睡眠模式，只有 Modern Standby（S0ix）可用，这个模式在 Windows 上表现不错，但 Linux 这边表现很差。

    但是也没办法说服 Linux，当时是陷入僵局了，然后偶然在 UAFB（Universal AMD Form Browser） 里打开了 AMD RAID Mode 选项，结果睡眠唤醒的速度就非常正常了，基本秒唤醒。

    在 UEFI 启动页面里，两块固态硬盘的显示也变成了 AMD RAID 0 和 AMD RAID 1 了，至少睡眠问题解决了（虽然不知道为什么）

    使用 HDMI 输出到外接显示器时颜色失真

    当时在使用 HDMI 输出到外接显示器时，发现颜色失真，整个画面看起来灰白发亮。

    理论上来说应该是 HDMI 的 RGB 色彩范围问题，但在 Windows 上是正常的并且可以切换 RGB 和 YCbCr 色彩范围的，Linux 这边在 KDE 显示设置中尝试切换色彩范围，但无论怎么切换都没有效果，颜色依然失真。

    排查半天无果，感觉也跟荣耀阉割的 BIOS 存在关系。

    最后使用 Steamdeck 的扩展坞接到电脑的 USB-C 接口上，使用 DisplayPort 输出到外接显示器，颜色就正常了，此时 Linux 识别为 DP 输出，同时设置切换颜色范围也是正常的，就很...神奇了。

    结语

    总的来说，我不是很推荐使用 Linux 的用户购买荣耀的新电脑，它的 BIOS 阉割了很多东西，出问题的情况下很难解决，我有一个朋友在用老的荣耀 Magicbook，他没有遇到任何问题，这事就很...

    当然了，荣耀的电脑在 Windows 上表现还是不错的，荣耀的 Turbo 优化其实还是不错的。
