    Kimi·K2-Turbo

    前言

    在很多 Blog 交流群有时能看到有些博主们在商量自己网站在用什么 CDN，什么 OSS 存储啊，也互相夸赞你的站打开有多快，图片加载有多快等。

    对于 OSS 存储其实在个人以往的《自建一个 Fediverse 实例，我们需要准备什么？》文章中对此已经略有提及，也要感谢猫猫博客站长 Firefly 的博文整理资料参考。

    自建一个 Fediverse 实例，我们需要准备什么？
    /2025/06/self-fediverse-prepare

    不过比起当时的整理来说，目前已经有了些许形势变化，所以还是可以再来回过头看看各家存储的新变化的。

    须知

    本文介绍以下情况：

    • 假设用户使用 100GB 标准非归档等存储，会列出选择购买对应存储资源包的情况
    • 假如出站流量也为 100GB 的情况，也同样列出购买资源包的情况
    • 对于国内服务商，也会尽量介绍中国大陆和其他地区计费情况，单位会使用人民币（￥），对于海外服务商则以他们在定价页面标出的货币为准，毕竟汇率也是具有时效性的。
    • 也可能包括其他优惠可能性等列出，但可能具有时效性
    • 所有计费等资料均以本文成文时最新资料为准，后续更新不一定及时，个人有更新也会及时在文章内标示。
    • 如果对象存储方案提供商提供的起步方案高于 100GB，则会灵活变化。
    • 不列出国内服务商按量计费价格，因为各家政策和实际使用情况实在是乱糟糟，海外按情况给出。

    有人可能会问：“你 100GB 是不是设的门槛太高了？很多个人博客可能压根到不了 100GB。”

    实话说这已经是个人感觉比较好计算（各家基本都有 100GB 标准，低于 100GB 就乱七八糟的了）和未来扩张性比较灵活的一个门槛了，如果你的需求更低还是建议你自行查看他们相关文档。

    注意下文购买资源包的价格都是一个月的情况，且本文只列出提供预付费资源包的价格，按量计费请查看官方文档或价格计算器。

    国内服务商

    曾经谈起国内 OSS 那是无不色变，“一夜一套房”的梦魇可以说缠绕很多包括我在内的站长许久，那么现在好了很多吗？好像也没有。

    实话说如果你不需要对存储有那么敏感的读写速度要求和稳定性，尤其是个人博客站长，还是适当委屈下访客，担心下自己的钱包选择海外对象存储厂商吧，国内的性价比还是难说的。

    尤其是现在 EdgeOne 免费后，有些站长使用它反代海外对象存储，理论上也算是一种好办法。

    123 网盘会员直链

    这种存储方式作为博客直链图床和文件存储的话，其实是一种很新颖的方式，它没有 OSS 式的管理方式（没有 S3），于个人来说性价比也很玄，取决于你对 123 网盘的粘性和会员选择，可以说是买 123 网盘会员的附赠品（只有会员才能使用直链功能和购买直链流量，一年有效期）

    我身边有朋友在使用这种方式作为站点图床，所以干脆也推荐一下吧，之前个人也对此有一定了解。

    功能来说也有 URL 和 Referer 鉴权，文件预览，IP 黑名单，自定义域名（增值服务，需备案，且 HTTPS 是按请求增值服务），静态网页托管，远程鉴权。

    个人推荐购买了 123 网盘会员的用户作为附赠甜品可以使用本功能，在保持会员身份的前提下直链价格还算可以，稳定性咱也不好说，不过 123 背后其实是全国也算有名的大厂，不一定很差。

    如果你是看上他们直链而想要购买的话...见仁见智吧，VIP 会员连续包月要 ￥25 一个月呢，如果对网盘使用需求没那么大不是很建议购买。

    你买了会员就是 20TB 云盘空间了（还能成长，每月成长 800GB，最高成长到 100TB），然后就可以额外购买直链流量包（实际上不单独购买也有 10GB/月的直链流量包，SVIP 会员则是 100GB/月直链流量包）

    直链流量包最低 1TB 起售，价格 ￥80，有效期一年，且需要是会员身份才能使用直链功能。

    海外服务商

    实话说如果真的没有硬性需求，海外其实挺合适了，毕竟很多都没有国内这么复杂的计费策略，也有很多都没有流出流量费用，所以海外这里我们在计算每个月 100GB 存储可能产生的成本时候就容易了不少。

    Cloudflare R2

    赛博活佛说是，当然 Cloudflare R2 在小需求上确实价格优势还不错，而且能享受 Cloudflare CDN 的便捷（需要域名托管在 Cloudflare）。

    兼容 AWS S3 的对象存储，基准功能不是很多，需要绑定域名配合 Cloudflare CDN 的策略配置食用，上限肯定是很高的。

    文档页面：https://developers.cloudflare.com/r2

    也有比较不错的免费额度（标准存储，不是归档存储）：

    类型额度
    存储额度10GB（所有存储桶的总额）
    Class A Operations每月前一千万次请求免费
    Class B Operations每月前一亿次请求免费
    出口流量免费

    实话说 R2 的请求额度还算慷慨，所以我们可以假设每月免费请求数没超额的情况下使用 R2 存储 100GB 数据的成本是 0.015 × 90（10GB 免费），为 1.35 美元每月。

    Blackblaze B2

    单论存储费用其实比 Cloudflare R2 价格低，每 GB 存储 0.006 美元，也有 10GB 的免费额度，但是出口流量直接用是不全免费的，免费流量为你的存储量三倍（例如存储了 100GB 数据，则出口 300GB 流量是免费的），超额后为每 GB 0.01 美元，并不便宜。

    但是有个东西叫带宽联盟，是 Cloudflare 发起的，真是赛博活佛了，如果你给它套上 Cloudflare CDN，对象存储的流量从 Cloudflare 流出，则是免费的。

    理论上是不是这样看比 Cloudflare R2 更划算？先别急着高兴，这就要请出第三个主角 ABC 请求了。

    虽然 B2 的 A 类请求（上传）是免费的，但关键的 B/C 类请求（下载和列出文件列表，分开计算）免费额度没有 Cloudflare R2 那么慷慨，甚至可以说少的可怜，仅有 2500 次每日，超出的 B/C 请求会按每 1000 次请求 0.0004 美元计费。

    所以你也能看出它的一个应用场景了：下载站和个人备份，这是典型低请求高流量高存储要求的情况，对于一部分下载站和个人用户使用来说，这个每日 2500 次的免费额度就稍微好过不少了，所以说如果有持续的超出 10GB 存储且每日读取和下载量不大，B2 看起来是一个比 R2 更为不错的选择。

    那么我们假设每日下载量没有超出免费额度，且存储 100GB 数据的话，成本就是 0.006 × 90（10GB 免费），为 0.54 美元每月，比起 Cloudflare R2 确实低不少。

    DigitalOcean Spaces

    兼容 AWS S3，他们的定价模式比较特殊，需要先购买基础订阅，每月 5 美元，可以拥有 250 GB 的存储空间和 1 TB（1024 GB）的出站流量，限制为 100 个存储桶。

    官方概述页面：https://www.digitalocean.com/products/spaces

    超额后的价格对比前两家是毫无吸引力的：0.02 美元每 GB 存储（标准存储），0.01 美元每 GB 流量（标准存储）

    不过这家没有什么 ABC 请求次数费用计费，这样来看对于大量的高请求次数，低流量消耗的小文件性价比上可能就有些微优势了。

    但是首先你需要真的有这需求，不然其他两家按存储价格还是把它按地上打。

    Hetzner S3 Storage

    兼容 AWS S3，由大名鼎鼎的德国 Hetzner（人称欧洲阿里云）提供的对象存储服务和 DigitalOcean Spaces 一样，都是先以订阅为基础，然后额外服务。

    官方概述页面：https://www.hetzner.com/storage/object-storage/

    它的价格是每月 4.99 欧元（欧元），包含 1TB 存储和 1TB 流量，流量超出 1 TB 后，按每 TB 1.00 欧元收费，存储超出 1TB 后，按每 TB 小时 0.0067 欧元收费（一天就是约 0.1608 欧元）。

    也和 DigitalOcean Spaces 一样，也同样没有什么 ABC 请求次数收费。

    看需求吧，这个情况和 DigitalOcean Spaces 一样可以说。

    Wasabi

    和前俩一样的基础订阅为基础，Pay as You Go，从 6.99 美元基础订阅套餐开始，提供 1TB 的基础存储，即使只存储了几十 GB，也会收取相当于 1 TB 存储容量的最低费用。

    Wasabi 官网：https://wasabi.com

    但是极具吸引力的一点它是没有出口流量和 API 请求费用，属于是把前俩先按地上打了，甚至某种程度上在大规模存储上比前面这几个都强。
